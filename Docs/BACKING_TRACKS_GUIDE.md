#

 Backing Track Playback System

Complete guide to Lyra's professional backing track system for solo performers and bands.

## Overview

Lyra's backing track system allows you to:
- **Play audio** while displaying chords and lyrics
- **Mix multiple tracks** with individual volume controls
- **Sync playback** with autoscroll
- **Add audio markers** for cue points
- **Route audio** to different outputs (click to in-ears, backing to PA)
- **Apply effects** (EQ, compression, fades)
- **Control with pedals** hands-free during performance

Perfect for solo performers, worship teams, theater productions, and bands.

## Supported Audio Formats

- **MP3**: Universal, compressed (recommended for most users)
- **M4A/AAC**: Apple format, excellent quality
- **WAV**: Uncompressed, highest quality, large files
- **AIFF**: Uncompressed, professional studio format
- **CAF**: Core Audio Format
- **FLAC**: Lossless compression (future support)

**Recommendation**: Use MP3 (320kbps) or M4A for balance of quality and file size.

## Playback Controls

### Basic Controls

**Play/Pause** (Space or foot pedal)
- Starts/pauses all tracks simultaneously
- Resumes from current position
- Tracks stay in sync

**Stop**
- Stops all tracks
- Returns to beginning (0:00)
- Clears reached markers

**Seek Bar**
- Drag to any position
- Tap to jump
- Displays current time and total duration

### Advanced Controls

**Playback Speed** (0.5x - 2.0x)
```
Tap speed indicator â†’ Select speed
- 0.5x = Half speed (practice)
- 0.75x = 75% speed (learning)
- 1.0x = Normal (default)
- 1.25x = 125% speed
- 1.5x = 150% speed
- 2.0x = Double speed
```

**Loop Section**
```
1. Tap loop button
2. Set start marker (current position or manual)
3. Set end marker
4. Loop enabled
5. Track loops between markers
```

**Skip Forward/Backward** (10s default)
```
Double-tap right side = +10s
Double-tap left side = -10s
Configurable in settings (5s, 10s, 15s, 30s)
```

### Time Display Options

Toggle time display format:
- **Elapsed**: Shows current position (2:30)
- **Remaining**: Shows time left (-1:45)
- **Total**: Shows both (2:30 / 4:15)

## Audio Markers and Cues

Add markers to audio tracks for navigation and automation.

### Marker Types

**Cue Marker** (Red flag)
- General cue point
- Visual reminder
- No automatic action

**Section Marker** (Blue note)
- Marks start of song section
- Jumps to chord/lyric section
- Auto-scrolls to section

**Warning Marker** (Yellow triangle)
- Warns of upcoming change
- Key change, tempo change, ending
- Visual flash when reached

**Ending Marker** (Red stop)
- Marks end of song
- Prepares for finish
- Can trigger auto-advance to next song

**Loop Point** (Green repeat)
- Loop start/end markers
- Quick loop setup
- For practice and rehearsal

**Sync Point** (Purple sync)
- Synchronization reference
- Align with visual cues
- Calibrate autoscroll

### Adding Markers

**During Playback:**
```
1. Play track
2. When you reach desired position, tap "+" or "Add Marker"
3. Marker placed at current position
4. Name and configure
```

**Manual Placement:**
```
1. Pause playback
2. Seek to exact position
3. Tap "Add Marker"
4. Set marker properties
```

**Precise Timing:**
```
1. Markers â†’ Add Marker
2. Enter exact time (e.g., 1:23.45)
3. Set type and name
4. Save
```

### Marker Actions

Trigger actions when marker is reached:

**Jump to Section**
- Automatically scroll to specific song section
- Perfect for live cues

**Show Message**
- Display alert/notification
- Remind performer of upcoming change

**Trigger MIDI**
- Send MIDI message at marker
- Change patch on keyboard
- Trigger lighting cue

**Change Display**
- Switch display mode

**Advance Song**
- Auto-advance to next song in set

**Blank Screen**
- Temporarily blank displays

## Audio Effects

Basic effects to enhance your backing tracks.

### Volume Fades

**Fade In:**
- Gradual volume increase at track start
- 0-10 seconds duration
- Smooth, professional entrance

**Fade Out:**
- Gradual volume decrease at track end
- 0-10 seconds duration
- Smooth ending

**Setup:**
```
Track settings â†’ Fades
Fade In: [slider] 0-10s
Fade Out: [slider] 0-10s
```

### Crossfade Between Tracks

Smooth transition from one track to next song.

**Enable:**
```
Settings â†’ Audio â†’ Crossfade
Duration: 2-8 seconds (default: 3s)
Curve: Equal Power (recommended)
```

**Behavior:**
- Last few seconds of Track 1 fade out
- First few seconds of Track 2 fade in
- Overlapping fade
- Seamless transition

**Curves:**
- **Linear**: Straight fade
- **Equal Power**: Natural sounding (recommended)
- **Exponential**: Fast then slow
- **S-Curve**: Slow-fast-slow

### EQ (Equalizer)

3-band EQ per track for tone shaping.

**Bands:**
- **Bass**: 100 Hz, -12 to +12 dB
- **Mid**: 1000 Hz (adjustable), -12 to +12 dB
- **Treble**: 10,000 Hz, -12 to +12 dB

**Common EQ Settings:**

**Flat (Default):**
```
Bass: 0 dB
Mid: 0 dB
Treble: 0 dB
No tone coloration
```

**Bass Boost:**
```
Bass: +6 dB
Mid: 0 dB
Treble: -2 dB
Fuller, warmer sound
```

**Treble Boost:**
```
Bass: -2 dB
Mid: 0 dB
Treble: +6 dB
Brighter, crisper sound
```

**Vocal Enhancement:**
```
Bass: -3 dB
Mid: +3 dB (at 2-3 kHz)
Treble: +2 dB
Clarity for vocals
```

**Setup:**
```
Track settings â†’ EQ
Enable EQ toggle
Adjust sliders
Preview changes in real-time
```

### Compression (Optional)

Dynamic range compression for consistent volume.

**Settings:**
- **Threshold**: -30 to 0 dB (when compression starts)
- **Ratio**: 1:1 to 20:1 (how much compression)
- **Attack**: 0.001 to 0.1s (how fast compression starts)
- **Release**: 0.01 to 1.0s (how fast compression stops)
- **Makeup Gain**: 0 to +12 dB (compensate for volume loss)

**Presets:**

**Gentle:**
```
Threshold: -15 dB
Ratio: 2:1
Subtle, transparent compression
```

**Moderate:**
```
Threshold: -20 dB
Ratio: 4:1
Balanced compression
```

**Heavy:**
```
Threshold: -25 dB
Ratio: 8:1
Significant compression
```

**When to Use:**
- Track has wide dynamic range
- Some parts too quiet, others too loud
- Want consistent volume level

## Performance Tips

### Before the Gig

**1. Test All Tracks**
- Play through entire set with backing tracks
- Verify all tracks load correctly
- Check volume levels
- Test audio routing

**2. Check File Formats**
- Use consistent format (MP3 recommended)
- Verify all files playable
- Keep backup files

**3. Adjust Mixer**
- Set volume levels for venue
- Balance backing vs. live instruments
- Test EQ settings
- Save mixer preset

**4. Set Up Markers**
- Add cue markers for key changes
- Mark tricky sections
- Add warnings for endings
- Test marker navigation

**5. Configure Routing**
- Test in-ear monitor feed
- Verify PA output
- Check for audio dropouts
- Test with actual hardware

### During Performance

**1. Start Cleanly**
- Cue track to beginning
- Verify correct song loaded
- Check volume before starting
- Count in band if needed

**2. Monitor Levels**
- Keep eye on volume meters
- Adjust if needed mid-song
- Watch for clipping (red)
- Balance with live sound

**3. Use Markers**
- Jump to markers for repeats
- Use cues for complex sections
- Navigate quickly if needed

**4. Emergency Controls**
- Know how to quickly stop
- Have blank screen pedal ready
- Practice emergency procedures
- Keep backup plan

**5. Smooth Transitions**
- Use crossfade between songs
- Practice set transitions
- Time gaps appropriately
- Use auto-advance carefully

### Troubleshooting During Performance

**Track Won't Play:**
1. Check if track loaded (file icon)
2. Verify audio output connected
3. Check master volume not muted
4. Restart app if necessary

**Audio Cutting Out:**
1. Check Bluetooth stability
2. Reduce number of active tracks
3. Lower playback speed to 1.0x
4. Close other apps

**Out of Sync:**
1. Stop and restart track
2. Adjust sync offset
3. Manually sync scroll position
4. Disable autoscroll if needed

**Wrong Track Playing:**
1. Stop immediately
2. Load correct song
3. Restart track
4. Check set list order

## Best Practices

### File Management

**Naming Convention:**
```
SongName_Backing.mp3
SongName_Click.mp3
SongName_Vocals.mp3
```

**Organization:**
- Keep all tracks for one song together
- Use consistent naming
- Include version numbers if multiple versions
- Date backups regularly

**File Size:**
- MP3 at 192-320 kbps is fine for most uses
- WAV for studio/highest quality
- Compress long tracks to save space
- Keep original uncompressed versions as backup

### Audio Quality

**Recording:**
- Record backing tracks at high quality
- 44.1 kHz or 48 kHz sample rate
- 24-bit depth if possible
- Use professional tools

**Export:**
- Export at matching sample rate
- Use high bitrate compression (256-320 kbps MP3)
- Normalize volume levels
- Add fade in/out during export if possible

**Testing:**
- Test on actual performance device
- Listen with performance headphones/monitors
- Check on different speakers
- Verify no audio artifacts

### Volume Levels

**Mixing:**
- Leave headroom (-3 to -6 dB peak)
- Balance all tracks
- Don't rely on limiter/compression
- Test at performance volume

**Live Performance:**
- Set backing tracks 10-20% quieter than live instruments
- Adjust per venue acoustics
- Consider crowd noise
- Have volume pedal ready for quick adjustments

### Backup Strategy

**Always Have Backup:**
- Export audio files to Files app
- Keep copy in cloud (iCloud, Dropbox)
- Have backup device with files
- Print emergency chord charts

**Pre-Load:**
- Load all backing tracks before performance
- Test each one briefly
- Verify routing
- Close unnecessary apps

## FAQ

**Q: Can I record backing tracks in Lyra?**
A: Future feature. Currently import pre-recorded files.

**Q: What's the maximum number of tracks?**
A: Technically unlimited, but 4-6 tracks recommended for performance stability.

**Q: Does this work offline?**
A: Yes, once tracks are imported, no internet needed.

**Q: Can I use streaming audio (Spotify, Apple Music)?**
A: No, due to DRM restrictions. Must use your own files.

**Q: Will tracks sync across devices via iCloud?**
A: Yes, if iCloud sync enabled for Lyra.

**Q: What if my backing track is slightly different tempo than chart?**
A: Use playback speed control (0.5x - 2.0x) to match.

**Q: Can I edit audio in Lyra?**
A: Basic trimming planned. For full editing, use DAW then import.

**Q: Does this drain battery?**
A: Audio playback uses moderate battery. Keep device charged or use power bank.

**Q: Can I use with Bluetooth speakers?**
A: Yes, but expect slight latency (100-200ms). Wired recommended for critical timing.

**Q: What about latency with multiple outputs?**
A: Depends on hardware. Professional audio interfaces have lower latency.
